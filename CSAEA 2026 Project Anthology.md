# Project Descriptions

## 1–5 Hours — Micro Projects

---

### Caesar Cipher Encoder/Decoder

A command-line program that shifts each character in a plaintext string by a user-specified integer key and reverses the operation for decryption. Requires Python 3.x or Java SE 17+; no external libraries — only standard string manipulation and modular arithmetic over the ASCII/Unicode range.

### Unit Conversion CLI Tool

A terminal application that converts between units of length, mass, temperature, and volume using hard-coded conversion factors. Requires Python 3.x or Java SE 17+ with the standard library only; no external dependencies or hardware.

### Number Guessing Game

A loop-driven console program that generates a pseudorandom integer within a defined range and prompts the user to guess, providing higher/lower feedback each iteration. Requires Python 3.x (`random` module) or Java SE 17+ (`java.util.Random`); no external libraries or hardware.

### Basic Calculator GUI

A windowed application with digit buttons, operator buttons, and a display field that evaluates arithmetic expressions in real time. Requires Python 3.x with `tkinter` (bundled with standard CPython distributions) or Java SE 17+ with Swing/JavaFX; no additional hardware.

### Markdown to HTML Converter

A script that parses a subset of Markdown syntax — headings, bold, italic, links, code blocks — and outputs a valid HTML file. Requires Python 3.x with the `re` module (standard library) or a lightweight third-party library such as `markdown2`; no hardware beyond a development workstation.

### LED Blink Patterns

An Arduino sketch that drives one or more LEDs through programmable blink sequences using `digitalWrite()` and `delay()` or timer interrupts. Requires an Arduino Uno R3 (ATmega328P, 5 V logic), standard 5 mm LEDs (forward voltage ~2 V), 220 Ω current-limiting resistors (one per LED), a USB-B cable for upload, and a breadboard with jumper wires. Current draw is approximately 15–20 mA per LED at 5 V.

### Password Strength Checker

A program that evaluates a candidate password against length, character-class diversity, and entropy thresholds, returning a strength rating. Requires Python 3.x (`re`, `math` from the standard library) or Java SE 17+ (`java.util.regex`); no external libraries or hardware.

### Dice Roll Simulator

A console or GUI application that simulates rolling N dice with M sides, aggregates results, and optionally displays frequency distributions. Requires Python 3.x (`random`, optionally `matplotlib` for histograms) or Java SE 17+; no hardware.

### Temperature Logger

A Raspberry Pi script that reads ambient temperature from a sensor at timed intervals and appends timestamped readings to a CSV file. Requires a Raspberry Pi 3B+ or later (5 V / 2.5 A micro-USB or USB-C power supply), a DS18B20 one-wire digital temperature sensor (3.3 V logic, operating range −55 °C to +125 °C), a 4.7 kΩ pull-up resistor on the data line, and three female-to-female jumper wires connecting VCC, GND, and data to GPIO 4. Software: Raspberry Pi OS, Python 3.x with the `w1thermsensor` library.

### Palindrome & Anagram Detector

A program that accepts two strings and determines whether one is a palindrome and whether the pair are anagrams of each other via character-frequency comparison. Requires Python 3.x or Java SE 17+ with the standard library only; no external dependencies.

### Pi-Hole Ad-Blocking DNS Server

A Raspberry Pi configured as a network-level DNS sinkhole that filters advertising and tracking domains for all devices on the LAN. Requires a Raspberry Pi 3B+ or later (5 V / 2.5 A power supply), a microSD card (8 GB minimum) with Raspberry Pi OS Lite, an Ethernet connection to the local router, and the Pi-hole open-source software installed via its `curl`-based installer. The router's DHCP settings must be reconfigured to point client DNS to the Pi's static IP address.

## 5–10 Hours — Short Projects

---

### Personal Portfolio Website

A static multi-page site showcasing projects, a résumé, and contact information, deployed to a free hosting platform. Requires HTML5, CSS3, and optionally vanilla JavaScript; deployment via GitHub Pages or Netlify (Git and a GitHub account required). No server-side runtime or specialized hardware.

### To-Do List App with File I/O

A console or GUI application that creates, reads, updates, and deletes tasks persisted to a local JSON or plain-text file. Requires Python 3.x (`json` module, `tkinter` for optional GUI) or Java SE 17+ (`java.io` / `java.nio`); no database or network access needed.

### Hangman with Word Bank

A text-based or graphical game that selects a random word from a bundled dictionary file and tracks incorrect guesses with a lives counter. Requires Python 3.x (`random`, optionally `tkinter` or `pygame` for graphics) or Java SE 17+; word bank stored as a flat text file — no external API.

### Arduino Ultrasonic Distance Sensor Display

An Arduino system that measures distance via an ultrasonic transducer and outputs the reading to a character LCD or serial monitor. Requires an Arduino Uno R3, an HC-SR04 ultrasonic sensor (5 V operating voltage, trigger and echo pins), a 16×2 HD44780-compatible LCD with an I²C backpack (PCF8574 adapter, 5 V), a breadboard, and jumper wires. The HC-SR04 draws approximately 15 mA at 5 V; the LCD backlight draws approximately 20–40 mA.

### Basic Web Scraper

A script that fetches HTML from a target URL, parses the DOM, and extracts structured data — headlines, prices, table rows — into a CSV or JSON file. Requires Python 3.x with `requests` and `beautifulsoup4` (`pip install` required); no hardware. Ethical and legal compliance with `robots.txt` is a prerequisite.

### Student Grade Book CLI

A terminal application that stores student records (names, assignments, scores) in memory or a local file and computes weighted averages and letter grades. Requires Java SE 17+ (collections framework, file I/O) or Python 3.x; no database or network.

### Pi-Powered Digital Clock Display

A Raspberry Pi project that renders the current time on an attached display, updated every second. Requires a Raspberry Pi 3B+ or later (5 V / 2.5 A supply), an SSD1306 OLED display (128×64, I²C interface, 3.3 V logic — SDA to GPIO 2, SCL to GPIO 3), and four female-to-female jumper wires. Software: Python 3.x with `adafruit-circuitpython-ssd1306` and `Pillow`.

### Tic-Tac-Toe with AI Opponent

A two-player or single-player grid game where the computer opponent uses the minimax algorithm to select optimal moves. Requires Python 3.x (with `tkinter` for GUI) or Java SE 17+ (Swing/JavaFX); no external libraries beyond the standard distribution.

### Simple REST API Client

A program that sends HTTP GET/POST requests to a public REST API (e.g., OpenWeatherMap, JSONPlaceholder), parses the JSON response, and displays formatted output. Requires Python 3.x with `requests` and `json`, or Java SE 17+ with `java.net.http.HttpClient`; a free API key where applicable.

### Binary Search Tree Visualizer

A Java or Python application that inserts, searches, and deletes nodes in a BST and renders the tree structure graphically after each operation. Requires Java SE 17+ with Swing/JavaFX or Python 3.x with `tkinter` or `matplotlib`; no external hardware.

### RFID Door Lock System

An Arduino project that reads an RFID tag and actuates a servo or solenoid lock when the UID matches an authorized list stored in EEPROM. Requires an Arduino Uno R3, an RC522 RFID reader module (3.3 V logic, SPI interface on pins 9–13, powered from the Uno's 3.3 V pin), RFID key cards/fobs (13.56 MHz MIFARE), a micro servo (SG90, 5 V, ~250 mA stall current) or a 5 V solenoid lock (requires an external supply and a MOSFET switching circuit with flyback diode), and a breadboard with jumper wires.

### Weather Station with LCD Output

An Arduino system that reads temperature, humidity, and barometric pressure from onboard sensors and displays the data on a character LCD. Requires an Arduino Uno R3, a BME280 sensor breakout (I²C, 3.3 V logic — use a bidirectional level shifter or a 3.3 V-tolerant breakout), a 16×2 I²C LCD, a breadboard, and jumper wires. Total system current draw is under 100 mA from the Uno's 5 V rail.

### Soil Moisture Auto-Watering System

An Arduino project that reads a capacitive soil moisture sensor and activates a water pump via a relay when moisture drops below a threshold. Requires an Arduino Uno R3, a capacitive soil moisture sensor v1.2 (3.3–5 V, analog output), a 5 V single-channel relay module, a small submersible DC water pump (3–5 V, ~100–200 mA), an external 5 V power supply for the pump (do not power directly from the Uno), vinyl tubing, and a breadboard with jumper wires.

### Media Server with Plex or Jellyfin

A Raspberry Pi configured to serve locally stored media files to network devices via a browser-based interface. Requires a Raspberry Pi 4 Model B (4 GB RAM recommended, 5 V / 3 A USB-C power supply), a microSD card (32 GB minimum), an external USB hard drive or SSD for media storage, an Ethernet connection, and Raspberry Pi OS with either Plex Media Server or Jellyfin installed via their official Linux ARM packages.

## 10–25 Hours — Medium Projects

---

### Chat Application (Socket Programming)

A client-server messaging system where multiple clients connect to a central server over TCP sockets and exchange text in real time. Requires Python 3.x (`socket`, `threading` modules) or Java SE 17+ (`java.net.Socket`, `java.net.ServerSocket`); clients and server can run on a single machine via `localhost` or across a LAN. No external libraries or hardware.

### Inventory Management System

A multi-module application that tracks products with fields for name, quantity, price, and category, supporting CRUD operations and basic reporting. Requires Java SE 17+ (collections, file I/O or JDBC for SQLite) or Python 3.x with `sqlite3`; no hardware. Optionally includes a GUI via Swing, JavaFX, or `tkinter`.

### Arduino Line-Following Robot

A two-wheeled differential-drive robot that uses infrared reflectance sensors to detect and follow a dark line on a light surface. Requires an Arduino Uno R3, an L298N dual H-bridge motor driver module (operating voltage 5–12 V), two DC gearbox motors (6 V, ~150 mA each under load), a two-wheel robot chassis with caster, a TCRT5000-based IR sensor array (minimum 2 sensors, 3.3–5 V operating), and a 7.4 V LiPo battery pack or 4×AA holder at 6 V wired to the motor driver's power input.

### Weather Dashboard with API Integration

A web application that fetches real-time weather data from a RESTful API and renders it in a browser with charts and location search. Requires Python 3.x with Flask and `requests` (backend) plus HTML/CSS/JavaScript (frontend), or a pure-frontend approach using `fetch()` and Chart.js. An OpenWeatherMap free-tier API key is required. No hardware.

### Raspberry Pi Home Temperature Monitor

A persistent monitoring system that reads temperature and humidity at intervals, logs data to a database, and serves a web dashboard. Requires a Raspberry Pi 3B+ or later (5 V / 2.5 A supply), a DHT22 sensor (3.3–5 V, data pin to GPIO 4 with a 10 kΩ pull-up resistor), Python 3.x with `Adafruit_DHT`, Flask, and `sqlite3`. Sensor current draw is approximately 1.5 mA during reads.

### Sorting Algorithm Visualizer

An interactive graphical application that animates the step-by-step execution of comparison-based sorting algorithms — bubble, selection, insertion, merge, quick — on a randomized array. Requires Python 3.x with `pygame` or `tkinter`, or Java SE 17+ with Swing/JavaFX; no external hardware.

### Encryption/Decryption Toolkit

A program offering multiple cipher implementations — Caesar, Vigenère, AES-128/256 — with a unified interface for encrypting and decrypting user-supplied text or files. Requires Python 3.x with `pycryptodome` (for AES) or Java SE 17+ with `javax.crypto`; no hardware.

### Multi-Page Flask or Spring Boot Web App

A server-rendered web application with user authentication, form submission, and database-backed content across multiple routes. Requires Python 3.x with Flask, Jinja2, and SQLite (or PostgreSQL) for the Python stack, or Java SE 17+ with Spring Boot, Thymeleaf, and an embedded H2 or PostgreSQL database for the Java stack. No hardware beyond a development workstation.

### Raspberry Pi Security Camera

A motion-activated camera system that captures images or video when movement is detected and stores them locally or streams to a web interface. Requires a Raspberry Pi 3B+ or later (5 V / 2.5 A supply), a Raspberry Pi Camera Module v2 (connected via the CSI ribbon cable), Python 3.x with `picamera2` and `motion` or the MotionEyeOS distribution. System draw is approximately 1.5–2 A with the camera active.

### Networked Multiplayer Game (LAN)

A real-time multiplayer game — e.g., tank battle, co-op dungeon crawler — where two or more players connect over a local network via TCP or UDP sockets. Requires Java SE 17+ (Swing/JavaFX for rendering, `java.net` for networking) or Python 3.x with `pygame` and `socket`; all machines must be on the same subnet.

### Servo-Controlled Robotic Arm

An Arduino-driven multi-joint arm with three to four degrees of freedom, controlled via potentiometers or serial commands. Requires an Arduino Uno R3 (or Mega 2560 for additional PWM channels), three to four SG90 micro servos (5 V, ~250 mA stall current each), matching 10 kΩ potentiometers for analog control, an external 5 V / 2 A power supply for the servos (do not power all servos from the Uno's 5 V pin), a 3D-printed or laser-cut acrylic arm chassis, and a breadboard with jumper wires.

### IR Remote-Controlled Vehicle

An Arduino-based car that receives commands from a standard infrared remote and drives DC motors accordingly. Requires an Arduino Uno R3, a VS1838B IR receiver module (3.3–5 V, output to a digital pin), a generic NEC-protocol IR remote, an L298N motor driver module (operating voltage 5–12 V), two DC gearbox motors (6 V), a two-wheel chassis with caster, a 7.4 V LiPo battery or 4×AA battery holder, and the `IRremote` Arduino library.

## 25–50 Hours — Extended Projects

---

### Full-Stack CRUD Application

A complete web application with a JavaScript or Java frontend, a RESTful backend, and a relational database supporting create, read, update, and delete operations on a domain entity. Requires either React (Node.js 18+, Express) plus PostgreSQL, or Java SE 17+ with Spring Boot and PostgreSQL/H2. No hardware beyond a development workstation.

### Arduino-Based Home Automation Hub

A central controller that manages multiple peripherals — relay-switched outlets, temperature/humidity sensors, and a servo-actuated vent — via serial commands or a simple web interface. Requires an Arduino Mega 2560 (for additional I/O), a 4-channel 5 V relay module (each relay rated for 10 A / 250 VAC — mains wiring must be supervised by a qualified adult), a DHT22 sensor, a servo, an ESP8266 Wi-Fi module (3.3 V logic, requires a level shifter from the Mega's 5 V TX), and an external 5 V / 3 A power supply. Any project interfacing with mains AC voltage requires instructor supervision and compliance with local electrical codes.

### Raspberry Pi Network Intrusion Detection System

A Raspberry Pi running Suricata or Snort in IDS mode, monitoring mirrored network traffic and logging alerts to a local database with a web-based dashboard. Requires a Raspberry Pi 4 Model B (4 GB RAM minimum, 5 V / 3 A USB-C supply), a microSD card (32 GB minimum), a managed switch capable of port mirroring, two Ethernet connections (one mirrored span port, one management), and Raspberry Pi OS with Suricata, a lightweight Flask dashboard, and Python 3.x.

### Machine Learning Image Classifier

A convolutional neural network trained on a labeled image dataset (e.g., CIFAR-10) that classifies new images via a command-line or web interface. Requires Python 3.x with TensorFlow 2.x or PyTorch, NumPy, and Matplotlib; a CUDA-capable GPU is recommended but not required — CPU training is viable on small datasets at longer training times.

### E-Commerce Storefront Prototype

A web application simulating an online store with product listings, a shopping cart, user authentication, and a mock checkout flow. Requires a frontend (React or plain HTML/CSS/JS), a backend (Flask or Spring Boot), a relational database (PostgreSQL or SQLite), and a payment-simulation module (no live payment gateway). No hardware.

### Custom Programming Language Interpreter

A tree-walk interpreter that lexes, parses (recursive descent), and evaluates a toy language supporting variables, arithmetic, conditionals, and loops. Requires Java SE 17+ or Python 3.x; no external libraries beyond the standard distribution. The project produces a REPL executable from the command line.

### Pi-Powered Retro Game Console

A Raspberry Pi configured with RetroPie or Lakka to emulate classic game consoles, paired with USB controllers and an HDMI display. Requires a Raspberry Pi 4 Model B (4 GB RAM, 5 V / 3 A USB-C supply), a microSD card (32 GB minimum, Class 10), an HDMI cable and display, one or two USB game controllers, and the RetroPie image flashed via Raspberry Pi Imager. Legally obtained ROMs only.

### Cybersecurity Capture-the-Flag Platform

### School-Wide Network Monitoring Dashboard

A web-based dashboard that polls network devices via SNMP or ping, displays uptime, bandwidth utilization, and alert history, and sends notifications when thresholds are exceeded. Requires a dedicated Linux server or Raspberry Pi 4 (4 GB RAM), Python 3.x with Flask, a time-series database (InfluxDB or SQLite), Grafana for visualization, and SNMP access to managed switches/routers. Coordination with the district IT department is required.

### Raspberry Pi Computer Cluster

A Beowulf-style cluster of multiple Raspberry Pi boards performing parallel computation on a distributable workload (e.g., prime sieve, Monte Carlo simulation). Requires three to five Raspberry Pi 4 Model B units (4 GB RAM each, 5 V / 3 A USB-C supply per unit), a Gigabit Ethernet switch, Cat5e/Cat6 cables, a multi-port USB power supply or individual adapters, a shared NFS directory, and Python 3.x with `mpi4py`. Total cluster power draw at load is approximately 15–25 W per node.

### Multiplayer Online Game with Persistent State

A client-server game with user accounts, real-time networked gameplay, and server-side state persisted to a database between sessions. Requires Java SE 17+ or Python 3.x, a networking framework (Netty for Java, `asyncio` for Python), PostgreSQL, and a frontend client in JavaFX, Pygame, or a browser-based canvas. Cloud deployment on a VPS is optional.

### Full Penetration Testing Lab Environment

A virtualized network of intentionally vulnerable machines used to teach reconnaissance, exploitation, and reporting in a controlled setting. Requires a host machine with 16 GB RAM minimum, VirtualBox or VMware, Kali Linux VM, and two to three target VMs (Metasploitable 2/3, DVWA, OWASP WebGoat). An isolated virtual network adapter is mandatory — no connection to the school production network.

### Autonomous Drone Navigation System (Arduino + Pi)

A quadcopter that uses onboard sensors and a Raspberry Pi for high-level navigation while an Arduino handles real-time motor control via a flight controller. Requires a Raspberry Pi 4 (4 GB RAM), an Arduino Mega 2560, a quadcopter frame kit with four brushless DC motors (rated for the frame weight), matching ESCs (electronic speed controllers, one per motor), an 11.1 V 3S LiPo battery (2200 mAh minimum), a PX4- or ArduPilot-compatible flight controller, a GPS module (u-blox NEO-6M), and an ultrasonic or LiDAR sensor for altitude hold. FAA Part 107 or recreational flyer regulations must be observed.

### Machine Learning Recommendation Engine

A system that ingests user-interaction data, trains a collaborative filtering or content-based model, and serves real-time recommendations through an API. Requires Python 3.x with scikit-learn or Surprise, Flask for the API layer, PostgreSQL for the interaction database, and a dataset (e.g., MovieLens). A CUDA-capable GPU is beneficial for larger matrix factorization workloads but not required.

### Custom Operating System Kernel Module

A loadable Linux kernel module (LKM) written in C that implements a custom device driver, system call, or virtual filesystem, tested in a sandboxed virtual machine. Requires a Linux development environment (Ubuntu 22.04+), GCC, kernel headers matching the running kernel version, `make`, and VirtualBox or QEMU for safe testing. A VM is mandatory to prevent damage to the host system.

### District IT Asset Tracking System

A full-stack application with barcode/QR scanning that tracks hardware assets (laptops, projectors, peripherals) across multiple buildings, with check-out/check-in workflows and reporting. Requires a web frontend (React), a backend (Flask or Spring Boot), PostgreSQL, and a USB HID barcode scanner or a mobile device camera with a JavaScript QR library. Deployment to a district-accessible server or intranet is required.

### IoT Smart Classroom Environment

An integrated system of networked sensors and actuators that monitors and controls classroom temperature, lighting, and occupancy via a central web dashboard. Requires a Raspberry Pi 4 as the hub, multiple Arduino Uno nodes (each with a DHT22, a photoresistor, and a PIR motion sensor), ESP8266 or nRF24L01+ modules for wireless communication between nodes and hub, a 4-channel relay module for lighting/HVAC relay control (mains wiring requires qualified supervision), and a Flask-based dashboard on the Pi. Budget 5 V / 2 A per node.

A multi-stage compiler that lexes, parses, performs semantic analysis, and emits bytecode or transpiles to a target language (e.g., C, JavaScript) for a custom-designed toy language. Requires Java SE 17+ or Python 3.x; optionally ANTLR for parser generation in the Java path. The project produces a command-line executable and a suite of test programs in the source language.
