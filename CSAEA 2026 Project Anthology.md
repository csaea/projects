## 1–5 Hours — Micro Projects

### 0x00 · Caesar Cipher Encoder/Decoder

A command-line program that shifts each character in a plaintext string by a user-specified integer key and reverses the operation for decryption. Requires Python 3.x or Java SE 17+; no external libraries — only standard string manipulation and modular arithmetic over the ASCII/Unicode range.

### 0x01 · Unit Conversion CLI Tool

A terminal application that converts between units of length, mass, temperature, and volume using hard-coded conversion factors. Requires Python 3.x or Java SE 17+ with the standard library only; no external dependencies or hardware.

### 0x02 · Number Guessing Game

A loop-driven console program that generates a pseudorandom integer within a defined range and prompts the user to guess, providing higher/lower feedback each iteration. Requires Python 3.x (`random` module) or Java SE 17+ (`java.util.Random`); no external libraries or hardware.

### 0x03 · Basic Calculator GUI

A windowed application with digit buttons, operator buttons, and a display field that evaluates arithmetic expressions in real time. Requires Python 3.x with `tkinter` (bundled with standard CPython) or Java SE 17+ with Swing/JavaFX; no additional hardware.

### 0x04 · Markdown to HTML Converter

A script that parses a subset of Markdown syntax — headings, bold, italic, links, code blocks — and outputs a valid HTML file. Requires Python 3.x with the `re` module (standard library) or a lightweight library such as `markdown2`; no hardware beyond a workstation.

### 0x05 · LED Blink Patterns

An Arduino sketch that drives one or more LEDs through programmable blink sequences using `digitalWrite()` and `delay()` or timer interrupts. Requires an Arduino Uno R3, standard 5 mm LEDs (forward voltage ~2 V), 220 Ω current-limiting resistors (one per LED), a USB-B cable for upload, and a breadboard with jumper wires. Current draw is approximately 15–20 mA per LED.

### 0x06 · Password Strength Checker

A program that evaluates a candidate password against length, character-class diversity, and entropy thresholds, returning a tiered strength rating. Requires Python 3.x (`re`, `math`) or Java SE 17+ (`java.util.regex`); no external libraries or hardware.

### 0x07 · Dice Roll Simulator

A console or GUI application that simulates rolling N dice with M sides, aggregates results, and optionally displays frequency distributions. Requires Python 3.x (`random`, optionally `matplotlib` for histograms) or Java SE 17+; no hardware.

### 0x08 · Temperature Logger

A Raspberry Pi script that reads ambient temperature from a sensor at timed intervals and appends timestamped readings to a CSV file. Requires a Raspberry Pi 4/5 (5 V USB-C power supply), a temperature sensor (e.g., DS18B20; 3.3 V logic, −55 °C to +125 °C range), a 4.7 kΩ pull-up resistor on the data line, and three jumper wires connecting VCC, GND, and data to GPIO 4. Software: Raspberry Pi OS, Python 3.x with `w1thermsensor`.

### 0x09 · Palindrome & Anagram Detector

A program that accepts two strings and determines whether one is a palindrome and whether the pair are anagrams via character-frequency comparison. Requires Python 3.x or Java SE 17+ with the standard library only; no external dependencies.

### 0x0A · Pi-Hole Ad-Blocking DNS Server

A Raspberry Pi configured as a network-level DNS sinkhole that filters advertising and tracking domains for all devices on the LAN. Requires a Raspberry Pi 4/5 (5 V USB-C supply), a microSD card (8 GB minimum) with Raspberry Pi OS Lite, an Ethernet connection to the academy subnet, and Pi-hole installed via its `curl`-based installer. The router's DHCP settings must point client DNS to the Pi's static IP.

### 0x0B · Morse Code Translator with Buzzer

An Arduino sketch that converts user-input text (via serial monitor) into audible Morse code on a piezo buzzer, and optionally decodes buzzer timing back to text. Requires an Arduino Uno R3, a passive piezo buzzer (3–5 V), a 100 Ω resistor, a breadboard, and jumper wires. Total draw under 30 mA.

### 0x0C · Binary / Hex / Decimal Converter

A console application that converts integers between binary, hexadecimal, octal, and decimal representations, displaying each step of the conversion algorithm. Requires Python 3.x or Java SE 17+; no external libraries.

### 0x0D · Rock-Paper-Scissors with Statistics

A console game that tracks win/loss/draw statistics over multiple rounds and reports percentages and streaks at exit. Requires Python 3.x or Java SE 17+; no external libraries or hardware.

### 0x0E · Mad Libs Generator

A terminal program that reads a story template with tagged blanks, prompts the user for nouns, verbs, and adjectives, then outputs the completed story. Requires Python 3.x (string formatting, file I/O); no external libraries.

### 0x0F · Reaction Time Tester

An Arduino circuit that illuminates an LED at a random interval and measures the elapsed milliseconds until the user presses a tactile switch, displaying the result on a serial monitor or an OLED display (e.g., SSD1306 128×64, I²C). Requires an Arduino Uno R3, one LED, one 220 Ω resistor, a tactile pushbutton, a 10 kΩ pull-down resistor, and a breadboard.

### 0x10 · RGB LED Color Mixer

### 0x11 · Hash Digest Generator

A command-line tool that computes MD5, SHA-1, and SHA-256 digests of a user-supplied string or file and outputs them side by side for comparison. Requires Python 3.x (`hashlib`) or Java SE 17+ (`java.security.MessageDigest`); no external libraries.

Requires 5+ Arduino Uno units, nRF24L01+ modules, assorted sensors, a Raspberry Pi 4/5 gateway, and Python 3.x (Flask, SQLite). Custom mesh routing protocol is part of the project.

### 0xED · Custom Version Control System

A distributed version control system supporting init, add, commit, diff, branch, merge, and clone operations, with content-addressable object storage (SHA-256 hashed blobs). Requires Python 3.x or Java SE 17+ (`hashlib`, file I/O); no use of Git libraries — the entire system is hand-built.

### 0xEE · Pi-Based Digital Forensics Workstation

A Raspberry Pi configured as a portable forensics workstation with tools for disk imaging, file carving, metadata analysis, and timeline reconstruction, using open-source tools. Requires a Raspberry Pi 4/5 (8 GB RAM recommended), a USB write-blocker, external storage, and Raspberry Pi OS with Autopsy, Sleuth Kit, and `volatility`.

### 0xEF · Autonomous Drone Swarm Coordination

Multiple drones that communicate wirelessly and execute coordinated maneuvers — formation flight, search pattern coverage, or relay communication. Requires 2–3 quadcopter builds (see 0xC0/0xD3), each with a Pi and wireless link (Wi-Fi or nRF24L01+), a ground station, and custom coordination software. FAA rules apply.

### 0xF0 · End-to-End Encrypted Messaging Application

A messaging application implementing the Double Ratchet algorithm (or simplified equivalent) for end-to-end encryption, with key exchange, message authentication, and forward secrecy. Requires Python 3.x (`cryptography` library, Flask backend, React or `tkinter` frontend) or Java SE 17+; no hardware.

### 0xF1 · 3D-Printed Robotic Arm with ML Object Sort

A 5+ DOF robotic arm (3D-printed, servo-driven) with a Pi camera performing object classification via a trained model, autonomously sorting items by category. Requires Autodesk Fusion (arm design), a 3D printer, servo motors (MG996R or similar), an Arduino Mega, a Raspberry Pi 4/5 with Pi Camera, Python 3.x (TensorFlow Lite, OpenCV), and an external power supply.

### 0xF2 · Campus-Wide IoT Sensor Network

A deployed network of sensor nodes across multiple rooms or buildings, reporting environmental data to a centralized dashboard with historical analytics and alerting. Requires 10+ Arduino nodes with wireless modules (ESP8266 or nRF24L01+), assorted sensors, a Raspberry Pi 4/5 gateway, a database (InfluxDB or PostgreSQL), and Grafana. Deployment coordination with facilities required.

### 0xF3 · Custom Hypervisor or Container Runtime

A type-2 hypervisor or OCI-compatible container runtime that isolates processes using Linux namespaces and cgroups, with a CLI for image management and instance lifecycle. Requires Python 3.x or C on Linux (Ubuntu 22.04+); root privileges; tested in a VM. Deep understanding of Linux kernel interfaces required.

### 0xF4 · Red Team / Blue Team Cyber Exercise

A structured adversarial exercise where one team (Red) attacks an isolated lab network while the other (Blue) defends, monitors, and responds in real time, culminating in incident reports from both sides. Requires the 0xCC lab environment, Kali Linux (Red), Suricata/Snort + SIEM (Blue), and a neutral referee/instructor. Strict rules of engagement required.

### 0xF5 · Autonomous Agriculture Robot

A field robot that navigates planting rows using GPS and computer vision, monitors plant health via camera and soil sensors, and reports data to a dashboard. Requires a Raspberry Pi 4/5 (vision + navigation), an Arduino Mega (motors + sensors), a Pi Camera Module, a GPS module (e.g., u-blox NEO-6M), soil sensors, DC motors with encoders, a rugged chassis (3D-printed or metal), and a battery pack.

### 0xF6 · Distributed Peer-to-Peer Chat System

A decentralized messaging system with no central server, using a DHT (distributed hash table) for peer discovery and end-to-end encryption for message confidentiality. Requires Python 3.x (`asyncio`, `cryptography`) or Java SE 17+; runs across multiple machines on the academy subnet.

### 0xF7 · Custom GPU Shader Pipeline (Software)

A software rasterizer implementing vertex transformation, clipping, rasterization, and fragment shading with a programmable shader interface, rendering 3D meshes to a framebuffer. Requires Java SE 17+ or Python 3.x (`numpy`, `Pillow` for output); no hardware GPU access — all rendering is CPU-based.

### 0xF8 · Pi-Based SCADA Security Testbed

A simulated industrial control system with a Raspberry Pi acting as a PLC simulator, an Arduino driving physical actuators (motors, valves/solenoids, indicator LEDs), and Modbus TCP communication, used to teach ICS/SCADA security assessment. Requires a Raspberry Pi 4/5 (PLC simulator + HMI), an Arduino Mega (I/O), relay modules, a motor, indicator LEDs, Python 3.x with `pymodbus`, and network isolation.

### 0xF9 · Secure Electronic Voting System

A web-based voting application demonstrating ballot secrecy, voter authentication, tamper-evident audit logs, and end-to-end verifiable tallying. Requires React (frontend), Flask or Spring Boot (backend), PostgreSQL, and a cryptographic library for commitment schemes. No hardware.

### 0xFA · CanSat Competition Satellite

A soda-can-sized satellite payload that records telemetry (altitude, temperature, pressure, GPS) during a rocket or drone launch, transmits data to a ground station, and is recovered under parachute. Requires an Arduino Mega or Teensy (flight computer), a pressure/temperature sensor (e.g., BMP280), a GPS module (e.g., u-blox NEO-6M), a radio transmitter (e.g., LoRa RFM95W, 915 MHz), a 3D-printed canister and parachute housing, a LiPo battery, and a ground station Raspberry Pi with matching LoRa receiver. Compliance with CanSat competition rules and launch-site safety procedures required.

### 0xFB · Self-Balancing Robot

A two-wheeled inverted-pendulum robot that maintains upright balance using PID control and an IMU (e.g., MPU-6050, I²C). Requires an Arduino Mega 2560 (or Uno with careful memory management), one IMU, a motor driver (e.g., L298N), two DC gearbox motors with encoders, a 3D-printed or metal frame, and a 7.4 V LiPo battery. PID tuning is the core challenge.

### 0xFC · Custom Network Protocol Suite

A from-scratch protocol stack implementing a custom physical-layer framing (serial or nRF24L01+), a network layer with addressing and routing, a transport layer with reliability, and an application-layer file transfer protocol, tested between two Arduino/Pi nodes. Requires two Raspberry Pi or Arduino units, nRF24L01+ or serial links, and Python 3.x or C; all protocol layers are hand-designed.

### 0xFD · Automated CI/CD Pipeline Platform

A self-hosted continuous integration / continuous deployment platform that monitors a Git repository, runs tests on push, builds artifacts, and deploys to a staging server, with a web dashboard showing build history and status. Requires a Raspberry Pi 4/5 (or Linux server), Docker, Python 3.x (Flask for the dashboard), Git hooks, and the academy's subnet for deployment targets.

### 0xFE · Digital Twin of a Physical System

A software simulation that mirrors a physical system — a 3D-printed robot arm, a greenhouse, or a sensor network — in real time, receiving live sensor data and visualizing the system's state in a 3D or 2D model. Requires the physical system (any instrumented project from this anthology), a Raspberry Pi 4/5 or workstation, Python 3.x (Flask, `matplotlib` or `three.js` for visualization), and serial or network data ingestion.

### 0xFF · Academy Capstone Portfolio Platform

A full-stack web application serving as the academy's permanent project showcase — students upload project write-ups, code repositories, photos, and demo videos; visitors browse by year, course, and category; an admin panel manages content. Requires React (frontend), Flask or Spring Boot (backend), PostgreSQL, file/media storage, deployment on a district-accessible server, and integration with the academy's authentication system.
